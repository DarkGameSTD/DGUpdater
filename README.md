# DGUpdater

A lightweight Windows updater designed to keep applications up-to-date by comparing local file hashes against a remote manifest and downloading only the files that have changed.

DGUpdater is intended for projects that require a simple, reliable update mechanism without downloading the entire application every time.

---

## Features

- 🔍 Verifies local files using **MD5 hashing**
- 📦 Downloads only missing or outdated files
- 📁 Automatically recreates missing directory structures
- 📊 Displays real-time download progress
- 🔄 Supports automatic self-updating
- 🚀 Launches the main application after a successful update
- 🔒 Requires Administrator privileges for safe file replacement
- ⚡ Uses asynchronous downloads for improved responsiveness

---

## How It Works

1. Calculates MD5 hashes for local application files.
2. Downloads a remote JSON manifest containing expected hashes and file locations.
3. Compares local files against the manifest.
4. Detects:
   - Missing files
   - Modified files
5. Downloads only the required files.
6. Replaces outdated files.
7. If the updater itself has been updated, it safely replaces itself using a temporary executable and batch script.
8. Launches the main application when the update has completed.

---

## Self-Updating

DGUpdater is capable of updating itself.

Since a running executable cannot replace itself, the updater:

- Downloads the new updater as a temporary executable.
- Generates a temporary batch script.
- Exits.
- Replaces the old updater.
- Starts the new version automatically.
- Cleans up temporary files.

This allows future improvements to the updater without requiring manual installation.

---

## File Verification

The updater verifies file integrity using **MD5** hashes.

Each file listed in the remote manifest contains:

- File hash
- Relative destination path

Only files whose hashes differ from the local version are downloaded.

## Update Process

```
Local Files
      │
      ▼
Generate MD5 Hashes
      │
      ▼
Download Remote Manifest
      │
      ▼
Compare Hashes
      │
      ├── No Changes
      │       │
      │       ▼
      │    Exit
      │
      ▼
Download Changed Files
      │
      ▼
Replace Files
      │
      ▼
Self-Update (if required)
      │
      ▼
Launch Application
```

---

## Notes

- Certain file types are intentionally ignored during hash generation.
- Directory structures are recreated automatically before downloading files.
- Running instances of the target application are closed before updating to avoid file access conflicts.
- Progress is displayed for every downloaded file.
